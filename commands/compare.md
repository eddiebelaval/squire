# /compare - Compare Changes

You are helping visualize what changed. Make it clear and actionable.

1. **Determine Comparison**
   Ask what to compare:
   - Current changes vs last commit
   - Current branch vs main
   - Two specific commits
   - Before/after a specific file

2. **Show Summary Stats**
   ```
   📊 Changes Summary:
   - Files changed: [count]
   - Lines added: [count]
   - Lines removed: [count]
   - Net change: [+/- count]
   ```

3. **Categorize Changes**
   Group by type:

   **🎨 UI/Frontend**
   - [List files]

   **⚙️ Logic/Backend**
   - [List files]

   **🗄️ Database**
   - [List files]

   **📝 Documentation**
   - [List files]

   **✅ Tests**
   - [List files]

4. **Highlight Key Changes**
   For each important file:
   - What changed in plain English
   - Why it matters
   - Any potential issues

5. **Visual Diff** (if requested)
   - Show specific file changes
   - Explain what the changes do
   - Use simple language

6. **Impact Assessment**
   - Breaking changes? (yes/no)
   - Affects existing features? (list)
   - New dependencies? (list)
   - Performance impact? (better/worse/neutral)

**Output Style:**
- Visual and organized
- Plain English explanations
- Focus on WHAT changed and WHY it matters
- Flag any concerns or risks
